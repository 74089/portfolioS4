Architectuur

Arduino Breadboard draadjes tekenen

Bij ons proftaak project maken we een website waarbij kinderen probleem oplossend leren denken en ze maken kennis met de basis van Arduino.

Het gedeelte dat ik programmeer is gebaseerd op onze user story: “Als leerling wil ik zien hoe ik de draadjes op mijn Arduino moet plaatsen zodat ik niet”. Ik heb dit berijkt door op een afbeelding van een breadboard een canvas te zetten. Op het canvas kan je rechte lijntjes trekken.

Getekende lijntjes op een canvas

Mogelijke uitbreidingen zouden nog zijn dat de lijntjes alleen vanuit zwarte puntjes naar zwarte puntjes kunnen getrokken worden en een button die de laatste lijn en alle lijntjes weghaalt.

GitHub Repo

Dit is het eerste project dat ik gemaakt heb waarbij ik het modulair probeerde te maken, ik heb dit gekregen door de hele tijd te denken: "Gaat dit ergens problemen veroorzaken als ik het ergens anders implementeer". Ook heb ik de basis van canvas geleerd, hierin had ik ook nog geen ervaring.

Arduino SVG

De vorige versie hiervan was via canvas, als ik het met canvas had gedaan was er geen makkelijke mogelijkheid om de lijntjes op de juiste punten te laten snappen.

Feedback 1

Misschien zou je naar SVG kunnen kijken en het via SVG punten aan elkaar verbinden.

Feedback 2

Op het moment dat ik deze feedback kreeg had ik een Arduino bordje als SVG maar nog geen functionaliteit. Ik had ook al geoefend met lijntjes trekken op een grid. https://jsfiddle.net/u8qy19ms/3/# Maar dit is later niet gebruikt.

Je kan makkelijk SVG manipuleren net zoals een canvas via een library die Raphaël heet.

Feedback 3

Op het moment dat ik deze feedback kreeg had ik het probleem dat de lijntjes niet op de punt van je muis kwamen terwijl dat wel de bedoeling was.

In de SVG had ik een viewbox gedefinieerd en die bleek mijn muis coördinaten helemaal verkeerd te pakken. Gelukkig wist mijn leraar welke woorden hij in stack overflow moest typen om een makkelijk antwoord te krijgen. https://stackoverflow.com/questions/10298658/mouse-position-inside-autoscaled-svg

Nadat dat geïmplementeerd was werkte de lijntjes hoe ik wilde maar de lijntjes kwamen achter het Arduino SVG’tje, dit is opgelost door de SVG in een functie te zetten en die aan te roepen voor de layer waar je al de lijntjes op tekent.

Wat heb ik er geleerd

• Met SVG’s werken

• SVG viewbox

• Paar functies in de Raphaël JS library

Conclusie

Er zijn meestal meerdere antwoorden mogelijk als het om programmeren gaat, maar om jezel het makkelijkste te maken moet je de simpelste weg nemen. De simpelste weg in dit geval was met een paar bochten waar ik nog geen ervaring mee had. Deze bochten, met name SVG en Raphaël, waren een zeer leerzame ervaring en het is goed dat ik weet dat ze bestaan en dat ik weet hoe ze gebruikt worden als ik ze nog een keer nodig heb.

Architectuur diagram

Programmeurs hebben architectuur diagrammen nodig om ideeën over het systeemstructuur en de gebruikerseisen te begrijpen, te verduidelijken en te communiceren. Het is een basiskader dat kan worden gebruikt in de ideation fase, waarmee collega’s de architectuur kunnen begrijpen, veranderingen kunnen bespreken en intenties kunnen communiceren.

Foutje

Doordat ik vroeg begonnen met prototypes te maken ben ik eigenlijk vergeten om een architectuur diagram te maken. Na een gesprek met mijn leraar had ik besloten om toch nog een architectuur diagram te maken. Dit heb ik gedaan zodat als er in de toekomst andere mensen aan het project moeten werken ze een duidelijk beeld hebben wat ze moeten programmeren en welke tools ze kunnen gebruiken en hoe ze met elkaar kunnen communiceren.

Eerste iteratie van het architectuur diagram

Arduino drawing module

De module die ik aanroepen waarin de kinderen hun opdrachten kunnen maken heet de “Drawing module”. Dit laadt in een div de arduino svg en breadboard, ook kunnen kinderen met knopjes selecteren of ze lampjes, draadjes of weerstandjes willen plaatsen.

Arduino assignment module

Om de kinderen te laten weten wat ze moeten doen wordt er uit de database gekeken bij welke opdracht ze zitten en die haalt vervolgens de stappen op die ze moeten doen. In de instellingen kunnen de kinderen ook hun taal instellen en dan pakt de database de stappen in de door hen gekozen taal.

Feedback Olaf

Er moet kort en duidelijk in het portfolio staan wat de 2 modules doen. Dit is toegepast door de tekst hierboven toe te voegen.

Een verbeter puntje zou zijn door bij de lijntjes uitleg te geven wat er precies met elkaar gecommuniceerd wordt. Deze tekst is toegevoegd bij de tweede iteratie.

Ook kreeg ik nog de vraag wat de 2 modules met elkaar doen waarop ik realiseerde dat ze met elkaar verbonden waren terwijl dat helemaal niet de bedoeling is. Dit is bij de tweede iteratie weg gehaald en de rest van het document is beter uitgelijnd.

Tweede iteratie van het architectuur diagram

Wat heb ik geleerd

Ik heb geleerd dat een goed architectuur diagram je helpt de applicatie simpel en snel te begrijpen. Door het te tekenen loop je vaak tegen bepaalde situaties aan waar nog niet over na gedacht was, waardoor je die situaties vervolgens weer kunt noteren en in de toekomst op voorbehoed bent. Als er met meerdere mensen aan een applicatie worden gewerkt is het goed om een duidelijk architectuur diagram te hebben zodat iedereen weet hoe de applicatie in elkaar komt te zitten.

Ik had deze opdracht samen met de persoon gemaakt waarmee ik ook aan het programmeren was. Ik had de Android drawing module uitgedacht en mijn klasgenoot de assignment module. Ik heb het eerst op papier gedaan en had het daarna via illustrator getekend. Naderhand kwam mijn klasgenoot en ik samen om hem via teams zijn eigen stuk er in te tekenen en alles samen te voegen.

Wat doe ik de volgende keer anders

De volgende keer zou ik eerder beginnen aan het maken van het architectuur diagram. Doordat ik het architectuur diagram zo laat hebben gemaakt ben ik er van bewust geworden hoe belangrijk een goed architectuur diagram is in de ideation fase. Het geeft zo veel duidelijkheid voor iedereen en je merkt bepaalde punten op die je anders helemaal over het hoofd zag.

De student heeft een applicatie gerealiseerd met een module pattern. Externe libraries zijn gebruikt om de applicatie te ontwerpen. De applicatie is tot stand gekomen via user stories en een schets van de architectuur.

Reflectie

Met Git heb ik al zeer veel ervaring. Voor dit semester had ik nog geen ervaring met GitLab. Zelf gebruikte ik altijd GitHub omdat ik daar de pro versie van heb door mijn vorige studie. Ik en mijn groepje moesten GitLab gebruiken omdat de docenten dat verplichten. Ik was op het begin geen fan van GitLab omdat ik er niet naar toe kan pushen vanuit mijn IDE. Dit leverde frustraties op. Ik had het opgelost door GitKraken te gebruiken op suggestie van een groepsgenoot van mij. Het probleem van GitKraken was dat het maar voor een week gratis was voor een company GitLab omgeving. Toen dit gebeurde was het al vrij laat in het semester dus ik besloot maar om alle bestanden te uploaden vanuit git.fhict.nl.

Conclusie

GitLab gebruiken kwam met wat moeite. In de toekomst zou ik gewoon terug gaan naar GitHub. Dit betekent niet dat als school me verplicht GitLab te gebruiken dat ik dit niet ga doen, maar als ik de vrijheid heb om te kiezen wat ik gebruik dan kies ik zelf GitHub.

Door al de moeite die bij GitLab kwam kijken heb ik er wel goed mee leren te werken.

De student gebruikt Git gedurende het hele traject. De code en Git-project is duidelijk beschreven volgens gangbare richtlijnen in de praktijk.

Ontwikkelomgeving

Op dit moment zit ik al 2 jaar op het HBO en hiervoor heb ik een 3 jarige MBO applicatie programmeur studie gedaan. Hierdoor heb ik al 5 jaar ervaring met ontwikkel omgevingen en code editors. Ik begon met een code editor genaamd codeLobster. Zelf voor de tijd waarin ik er mee begon te werken was de IDE al verouderd en in het tweede jaar kregen we de vrijheid om onze eigen IDE te kiezen.

Atom

Zelf gebruik ik de editor Atom, ik heb hier al bijna 4 jaar ervaring mee. Tijdens mijn vorige opleiding mochten we onze eigen editor gebruiken en toen kreeg ik Atom geadviseerd, door er meer en meer mee te werken begon ik het fijner en fijner te vinden. Atom heeft een ingebouwde Git functie waarbij je kunt inloggen met je GitHub Account en vervolgens makkelijk kunt pushen, pullen, commiten en van branch kunt wisselen. Er zijn ook heel veel packages gemaakt door de community (een package in Atom is het zelfde als een package in VScode). Een van die packages heet Git-log, als je het opent mag je een van je repos kiezen om vervolgens een overzichtelijke lijn van alle branches en commits te zien. Door deze package ben ik gestopt met GitKraken te gebruiken en volledig overgestapt op allen Atom. Ik heb ook nog veel meer packages die hier te zien zijn.
